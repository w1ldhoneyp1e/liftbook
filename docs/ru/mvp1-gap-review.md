# MVP 1 Gap Review

Дата обзора: 2026-04-30

Этот документ фиксирует, насколько текущая реализация Liftbook близка к MVP 1, и что еще остается добить перед финишем.

## Уже готово

- Mobile-first экран дня на iPhone-подобном viewport.
- Offline-first локальное хранение данных в IndexedDB.
- Гостевой аккаунт через backend.
- Базовый backup/sync через `push` и `pull`.
- Автосинк при появлении сети.
- Навигация по датам с текущей датой по умолчанию.
- Горизонтальная лента дат и календарь.
- Добавление упражнений по группам мышц и через поиск.
- Создание пользовательских упражнений.
- Переименование и удаление пользовательских упражнений.
- Добавление и редактирование подходов.
- Ввод веса и повторений с клавиатуры и кнопками `+ / -`.
- Поддержка `kg` и `lb`.
- Русский и английский UI.
- Стартовый каталог упражнений.
- Дефолты из прошлого результата.
- Таймер отдыха на экране текущего дня.
- Автозапуск таймера отдыха через настройку.
- Серверный `cursor` на sequence.
- Идемпотентный `push` на backend.
- Страничный `pull` с `hasMore`.
- PostgreSQL backend path.
- Lifecycle cleanup для sessions и sync events.

## Готово частично

- История.
  Сейчас есть только текущий экран дня. Отдельный history screen или компактная история по упражнениям еще не сделаны.
- Примерная длительность тренировки.
  Таймер отдыха есть, но отдельного пользовательского представления длительности тренировки пока нет.
- Визуальное разделение прошлого, сегодня и будущего.
  Состояния уже различаются, но это еще можно полировать.
- Calendar drawer.
  Календарь работает, но это еще не финальный polished UX.
- Sync status UI.
  Статус и счетчики есть, но это все еще инженерно ориентированный слой.

## Нужно позже

- Полноценная история и прогресс.
- Графики.
- Расширение каталога до 150 упражнений.
- Шаблоны тренировок.
- Полноценный multi-device sync.
- Конфликт-UI для пользователя.
- Корпоративная интеграция.
- Платежи.
- Production auth.

## Оставшийся MVP 1 scope

Для MVP 1 нам еще важно:

1. Пройти основной сценарий как пользователь и убрать шероховатости, мешающие тренировке.
2. Сделать финальный gap review pass по `requirements.md` и `ux-flow.md`.
3. Убедиться, что документация и поведение кода совпадают.

## Критерий готовности

MVP 1 можно считать готовым, когда:

- основной тренировочный flow проходит на мобильном экране без заметных UX-блокеров;
- офлайн сценарий работает;
- guest account и sync продолжают работать;
- backend smoke и build зеленые;
- оставшиеся gaps осознанно перенесены в MVP 2+.
