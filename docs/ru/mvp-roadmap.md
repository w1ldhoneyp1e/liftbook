# Дорожная карта MVP

## MVP 1: офлайн-дневник тренировок с аккаунтом

Цель: пользователь может провести реальную тренировку в зале с Liftbook на iPhone, полностью офлайн.

Текущий статус и оставшиеся gaps зафиксированы в [docs/ru/mvp1-gap-review.md](/home/kirill-yashmetov/projects/liftbook/docs/ru/mvp1-gap-review.md).

Основные возможности:

- Открытие как app-like mobile experience.
- Локальный/гостевой режим без входа.
- Account-ready путь для backup/sync.
- Опциональное создание гостевого аккаунта из настроек, если API доступен.
- Ручная синхронизация после создания гостевого аккаунта.
- Автоматическая синхронизация при появлении сети и автоматический `pull` изменений с других устройств.
- Навигация по датам с текущей датой по умолчанию.
- Записи тренировок без обязательного старта/финиша.
- Добавление упражнений в тренировку.
- Добавление через группы мышц и быстрый поиск по названию.
- Добавление подходов.
- Ввод веса и повторений.
- Поддержка kg и lb.
- Русский и английский UI.
- Около 50 стартовых упражнений по группам мышц.
- Создание пользовательских упражнений.
- Переименование и удаление пользовательских упражнений.
- Tracking modes на уровне упражнения.
- Просмотр прошлого результата по упражнению.
- Создание дефолтных подходов из прошлого результата.
- Настройка отключения дефолтов из прошлого результата.
- Таймер отдыха между подходами.
- Настройка автозапуска таймера, если стоимость реализации приемлема.
- Таймер отдыха на экране текущего дня.
- Примерная длительность тренировки по использованию таймера.
- Просмотр истории.
- Локальное хранение всех данных.

Вне рамок:

- Полный облачный sync, если аккаунт сначала будет только identity-слоем.
- Платежи.
- Социальные функции.
- Корпоративная интеграция.
- Режим тренер/клиент.
- Шаблоны тренировок.

## MVP 2: улучшение тренировочного процесса

Цель: уменьшить повторный ввод и ускорить частые сценарии.

Возможные функции:

- Шаблоны тренировок.
- Переиспользование прошлой тренировки.
- Расширение каталога до 150 упражнений.
- Заметки к упражнению или подходу.
- Редактирование завершенных тренировок.
- Дополнительная метаинформация подходов: warm-up, рабочий подход, отказ, RPE/RIR.

## MVP 3: прогресс и инсайты

Цель: сделать прогресс видимым.

Возможные функции:

- Личные рекорды.
- Экран истории упражнения.
- Графики объема.
- Расчетный 1RM.
- Сравнение недавних результатов.

## MVP 4: sync и несколько устройств

Цель: защитить данные и разрешить использование на нескольких устройствах.

Возможные функции:

- Cloud backup.
- Multi-device sync.
- Обработка конфликтов.
- Экспорт/импорт.

## MVP 5: готовность к интеграциям

Цель: подготовить Liftbook как сервис или модуль внутри более крупного продукта.

Возможные функции:

- Public/internal API.
- Модель организаций.
- Модель ролей.
- Шаринг данных по согласию.
- Integration events.
- Границы администрирования и владения данными.
