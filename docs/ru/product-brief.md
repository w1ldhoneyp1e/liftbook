# Краткое описание продукта

## Название

Liftbook

## Видение

Liftbook — быстрый, бесплатный, mobile-first дневник тренировок для людей, которые занимаются в зале и хотят удобно записывать упражнения, подходы, повторения и вес.

В будущем продукт может остаться самостоятельным сервисом или стать частью более крупного корпоративного HR/training-продукта.

## Взгляд заказчика

В первой версии нужно доказать, что Liftbook удобно использовать прямо во время реальной тренировки:

- пользователь открывает приложение на iPhone;
- добавляет упражнения;
- записывает подходы с весом и повторениями;
- видит или переиспользует прошлый результат по тому же упражнению;
- приложение работает без интернета.

## Целевая аудитория

- Обычные посетители зала, которым нужен простой дневник тренировок.
- Более продвинутые спортсмены, которым важны прошлые результаты и прогресс.

## Отличия

Многие трекеры тренировок платные или становятся платными для базового использования. Заметки и таблицы гибкие, но слишком медленные во время тренировки.

Liftbook должен быть:

- бесплатным для базового дневника тренировок;
- быстрее заметок и таблиц;
- надежным офлайн;
- сфокусированным на быстром вводе результата тренировки.

## Принципы продукта

- Mobile-first: основные экраны должны быть удобны на телефоне и одной рукой.
- Offline-first: дневник тренировок должен работать без интернета.
- Local-first data entry: ввод данных не должен ждать ответа сервера.
- Переводимый UI: весь пользовательский текст проходит через локализацию.
- Guest-friendly: пользователь может начать без входа.
- Free core: базовое логирование тренировок остается бесплатным.

## Будущий вариант

Liftbook может интегрироваться в крупный корпоративный продукт для развития и обучения сотрудников. Поэтому архитектура должна поддерживать:

- API-first интеграции;
- доменные границы;
- правила синхронизации и владения данными;
- будущие аккаунты и организации;
- возможное выделение доменов в отдельные сервисы;
- выборочный шаринг данных тренировок для достижений, челленджей и соревнований;
- доступ к персональным данным только по согласию пользователя.
