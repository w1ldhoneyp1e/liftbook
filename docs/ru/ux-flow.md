# UX flow MVP

## Основная навигация

Основная навигация идет по датам.

При открытии Liftbook приложение показывает текущую дату.

Экран текущей даты — основной рабочий экран во время тренировки.

UI должен ясно показывать, выбрана прошлая дата, сегодня или будущая дата.

Предпочтительное направление — сдержанное цветовое различие в шапке/области дат или тонком акценте. Tint всего экрана можно исследовать, но он не должен мешать сканировать поля тренировки.

Прошлые и будущие даты можно редактировать.

## Экран текущей даты

Экран текущей даты показывает:

- выбранную дату;
- компактную горизонтальную навигацию по датам;
- действие открытия календаря;
- список упражнений на дату;
- действие добавления упражнения;
- таймер отдыха текущего дня;
- доступ к настройкам или профилю.

Экран не требует формального “начать тренировку” или “закончить тренировку”.

## Добавление упражнения

Базовый сценарий:

1. Пользователь нажимает добавление упражнения.
2. Видит группы мышц.
3. Выбирает группу мышц.
4. Выбирает упражнение.
5. Упражнение добавляется к текущей дате.

Сценарий поиска:

1. Пользователь нажимает добавление упражнения.
2. Сразу может искать упражнение по названию.
3. Выбирает найденное упражнение.
4. Упражнение добавляется к текущей дате.

## Дефолты из прошлого результата

Когда упражнение добавляется и есть прошлые данные, Liftbook сразу создает редактируемые подходы из самого свежего прошлого результата.

Пример:

- Прошлый жим лежа: 60 kg x 8, 60 kg x 8, 60 kg x 7.
- Новая запись жима получает три редактируемых подхода с теми же весом и повторениями.

Это поведение настраивается.

Сгенерированные подходы сразу являются обычными записями. Подтверждение не нужно.

Дефолты из прошлого результата должны работать для прошлых, сегодняшней и будущих дат.

## Список упражнений

Экран активной даты показывает список упражнений.

Упражнения раскрыты по умолчанию.

Пользователь может:

- свернуть все упражнения;
- раскрыть все упражнения;
- свернуть или раскрыть одно упражнение.

Каждое раскрытое упражнение показывает свои подходы.

## Ввод подхода

Ввод подхода оптимизирован для быстрого использования в зале.

Для веса и повторений пользователь может:

- ввести значение с клавиатуры;
- увеличить или уменьшить значение кнопками.

Шаги по умолчанию:

- Повторения: 1.
- Вес в kg: 1 kg.
- Вес в lb: небольшой шаг, точное значение открыто.

MVP не требует checkbox completed или отдельного состояния done.

Каждое раскрытое упражнение показывает действие добавления подхода рядом с существующими подходами. Оно должно ощущаться как следующий потенциальный подход и быть похожим по размеру и форме на строки подходов.

## Пустой день

Пустой день должен быть визуально легким.

MVP показывает маленькое действие добавления упражнения. Более богатое пустое состояние можно спроектировать позже.

## Таймер отдыха

Таймер отдыха принадлежит экрану текущей даты.

Ручной таймер обязателен.

Автоматический таймер настраивается. Точный триггер открыт:

- после заполнения данных подхода;
- после создания нового подхода;
- после сохранения/подтверждения подхода.
