# MVP 2

MVP 2 нужен не для “большой платформы”, а для того, чтобы сделать повторяющиеся тренировки быстрее и удобнее.

## Сначала

Сначала закрываем живые замечания по интерфейсу, стабильности и DX.

Отдельно в ближайший UX-проход входит:

- определить итоговую палитру светлой и темной темы;
- синхронизовать эту палитру с иконкой приложения;
- после этого уже приводить к тем же цветам empty states, popup/drawer-слои и системные акценты.

## Основной фокус

1. уменьшить ручной повторный ввод;
2. ускорить сборку привычной тренировки;
3. дать полезную историю упражнения прямо во время тренировки;
4. не перегрузить продукт.

## Что входит в MVP 2

1. **История упражнения**
   - последние выполнения;
   - быстрый доступ из карточки упражнения;
   - практический акцент, без тяжелой аналитики.

2. **Переиспользование прошлой тренировки**
   - взять упражнения и подходы из прошлого дня;
   - быстро применить к новой дате.

3. **Шаблоны тренировок**
   - создать шаблон из текущего дня;
   - применить, редактировать, удалить.

4. **Заметки**
   - короткая заметка к упражнению;
   - короткая заметка к подходу.

5. **Метаинформация подхода**
   - warm-up / working set / failure-like markers;
   - только если это не ломает основной flow.

6. **Расширение каталога**
   - больше упражнений;
   - более аккуратные мышечные группы и названия.

## Что не входит

- большая аналитика;
- социальные функции;
- корпоративная интеграция;
- production-grade multi-device sync;
- платежи и роли.

## Порядок

1. история упражнения
2. переиспользование прошлой тренировки
3. шаблоны
4. заметки
5. метаинформация подходов
6. каталог
