# Спецификация MVP экрана дня

## Назначение

Экран дня — основной экран Liftbook. На нем пользователь записывает упражнения, подходы, вес и повторения для выбранной даты.

## Приоритеты layout

- Mobile-first.
- Достаточно плотный для зала.
- Удобный одной рукой.
- Минимальные пустые состояния.
- Нет обязательного start/finish действия.

## Область шапки

Шапка включает:

- текущую выбранную дату;
- компактную горизонтальную ленту дат;
- кнопку календаря;
- опциональный доступ к настройкам/профилю.

Приложение открывается на сегодняшней дате.

UI должен ясно показывать, выбрана прошлая дата, сегодня или будущая дата.

Предпочтительное направление — сдержанное цветовое различие:

- акцент цвета в дате/шапке;
- тонкий верхний или боковой акцент;
- мягкий tint шапки.

Tint всего экрана можно исследовать, но он должен оставаться сдержанным, чтобы поля тренировки легко читались.

Прошлые и будущие даты редактируемы.

Если выбранная дата не сегодня, UI может показывать действие возврата к сегодняшнему дню. Точное место открыто.

## Навигация по датам

Основная навигация:

- горизонтальная лента дат.

Вторичная навигация:

- календарь, открываемый с экрана дня.

Дополнительный helper:

- возврат к сегодняшнему дню, если выбрана другая дата.

## Область упражнений

Экран дня показывает список упражнений выбранной даты.

Одно и то же упражнение может встречаться на одной дате несколько раз.

Поведение по умолчанию:

- упражнения раскрыты;
- есть управление “свернуть все”;
- у каждого упражнения есть собственное управление collapse/expand.

## Элемент упражнения

Раскрытое упражнение содержит:

- название упражнения;
- группу мышц или небольшую метаинформацию, если полезно;
- управление collapse/expand;
- строки подходов;
- строку/действие добавления подхода.

## Строка подхода

Для MVP упражнений с весом и повторениями строка подхода поддерживает:

- ввод веса;
- ввод повторений;
- ввод с клавиатуры;
- кнопки увеличения/уменьшения.

Шаги по умолчанию:

- Повторения: 1.
- Вес в kg: 1 kg.
- Вес в lb: небольшой шаг, точное значение открыто.

MVP не требует отдельного состояния completed/done. Введенные или сгенерированные строки подходов считаются обычными записями тренировки.

## Добавление подхода

У каждого раскрытого упражнения есть собственное действие добавления подхода.

Оно должно находиться рядом с существующими подходами и выглядеть как потенциальный следующий подход похожего размера и формы.

## Пустой день

Если выбранная дата не содержит упражнений, экран остается простым и показывает небольшую кнопку добавления упражнения.

Более богатое пустое состояние можно спроектировать позже.

## Таймер отдыха

Таймер отдыха является частью экрана дня.

MVP требует ручной таймер.

Автоматический таймер настраивается и может быть реализован после выбора триггера.
