# Требования

## Подтвержденные требования

### Продукт

- Название продукта: Liftbook.
- Название репозитория: `liftbook`.
- Продукт в первую очередь является личным дневником тренировок.
- Первая полезная версия должна хорошо работать на iPhone.
- Базовое использование должно быть бесплатным.
- Подход: mobile-first.
- Подход: offline-first.
- Аккаунты нужны рано, но основное использование должно оставаться доступным офлайн.
- Пользователь может пользоваться приложением без входа.
- Все пользовательские тексты должны быть переводимыми.

### Основной пользовательский сценарий

Во время тренировки пользователь должен уметь:

- открыть Liftbook на iPhone;
- увидеть текущую дату по умолчанию;
- переключаться между записями по датам;
- добавлять записи тренировки без обязательного старта/финиша;
- добавлять упражнения;
- записывать подходы;
- вводить повторения;
- вводить вес;
- видеть прошлый результат по тому же упражнению;
- использовать прошлые подходы как значения по умолчанию, если данные есть.

### Офлайн

Все, что нужно для основного тренировочного сценария, должно работать без интернета:

- начало и продолжение тренировки;
- добавление упражнений;
- запись подходов, повторений и веса;
- просмотр прошлых данных по текущему упражнению;
- редактирование текущей тренировки;
- локальное сохранение данных;
- использование стартового каталога после установки/кеширования.

Первичная установка и первая авторизация могут требовать интернет. После этого базовый сценарий тренировки должен оставаться доступным офлайн.

### Синхронизация

- Данные сначала сохраняются локально.
- Синхронизация происходит позже, когда есть сеть.
- Модель синхронизации нужно проектировать рано, даже если серверный sync не входит в MVP 1.
- Account-backed sync желателен рано.
- Локальные данные остаются основным write path во время тренировки.
- До создания аккаунта должен быть гостевой/локальный режим.
- Аккаунт должен открывать backup и sync, а не блокировать базовое использование.

### Будущая интеграция

Продукт может стать частью корпоративного HR/training-продукта для обучения и развития сотрудников.

Корпоративным интеграциям могут понадобиться данные тренировок для достижений, соревнований или других opt-in механик.

### Каталог упражнений

- В MVP нужен стартовый каталог упражнений.
- Цель MVP: около 50 стартовых упражнений.
- После первого MVP желателен каталог примерно на 150 упражнений.
- Упражнения группируются по группам мышц.
- Начальные группы: грудь, спина, ноги, плечи, бицепс, трицепс, кор, ягодицы, кардио, все тело, другое.
- По умолчанию добавление упражнения начинается с группы мышц.
- Пользователь также должен сразу искать упражнения по названию.
- Упражнение имеет tracking mode, который управляет полями ввода.
- Пользовательские упражнения важны и должны попасть в MVP 1, если возможно, или сразу после него.

Возможные режимы:

- Вес и повторения.
- Повторения с собственным весом.
- Время.
- Дистанция и время.
- Дополнительный вес + собственный вес.

### Единицы и локализация

- MVP поддерживает kg и lb.
- MVP поддерживает русский и английский.
- Архитектура локализации должна позволять добавлять языки позже.
- Пользовательский текст должен меняться при переключении языка.
- Доменные значения в UI, например группы мышц и названия упражнений, должны быть готовы к локализации.

### Модель тренировки

- Тренировка не требует обязательного start/finish ритуала.
- Приложение работает как набор записей тренировок.
- Основная навигация идет по датам.
- При открытии показывается текущая дата.
- Навигация по датам включает компактную горизонтальную ленту.
- Пользователь может открыть календарь.
- Экран текущего дня показывает список упражнений.
- UI должен явно показывать, что выбранная дата не сегодня.
- Прошлые, сегодняшняя и будущие даты визуально различаются цветом.
- Цвет состояния даты должен быть сдержанным: в шапке, области дат или тонком акценте.
- Сильный tint всего фона можно исследовать, но базовое направление сдержанное.
- Прошлые даты можно редактировать.
- Будущие даты можно редактировать.
- UI может иметь быстрое возвращение к сегодняшнему дню.
- Точное место этой кнопки открыто.
- Упражнения раскрыты по умолчанию.
- Можно свернуть все упражнения.
- Можно свернуть отдельное упражнение.
- Можно запускать таймер отдыха между подходами.
- Примерную длительность тренировки можно считать от первого запуска таймера до последнего завершения.
- Автозапуск таймера настраивается.
- Точный триггер автозапуска открыт: после заполнения подхода или после создания/сохранения нового подхода.
- Таймер отдыха является частью экрана текущего дня.

### Дефолты из прошлого результата

- При добавлении упражнения Liftbook сразу создает редактируемые подходы из самого свежего прошлого результата, если он есть.
- Дефолты должны работать для прошлых, текущей и будущих дат.
- Сгенерированные подходы считаются обычными завершенными записями сразу, это не план и не требует подтверждения.
- В MVP нет обязательного подтверждения подходов.
- Поведение настраивается.

### Ввод подходов

- Ввод веса и повторений поддерживает клавиатуру.
- Ввод веса и повторений поддерживает кнопки плюс/минус.
- Шаг веса по умолчанию для kg: 1 kg.
- Шаг веса по умолчанию для lb должен быть небольшим; точное значение открыто.
- Шаг повторений: 1.
- MVP не требует отдельного состояния completed/done для подходов.
- У каждого раскрытого упражнения есть собственное действие добавления подхода.
- Действие добавления подхода должно быть рядом с существующими подходами и ощущаться как потенциальный следующий подход похожего размера и формы.

### Exercise entries

- Одно и то же упражнение может быть добавлено на одну дату несколько раз.

### Пустой день

- Пустой день должен быть визуально легким.
- MVP показывает пустой день с маленькой кнопкой добавления упражнения.
- Более богатое пустое состояние можно спроектировать позже.

### Корпоративный шаринг данных

- Данные тренировок могут передаваться корпоративной интеграции при необходимости.
- Передача должна быть основана на согласии пользователя.

## Открытые вопросы

### Scope продукта

- Нужны ли шаблоны тренировок с первого дня?
- Должно ли активное состояние тренировки переживать закрытие вкладки/браузера?
- Какие методы входа делать первыми: email/password, magic link, OAuth, passkeys?
- Должны ли пользовательские упражнения быть в MVP 1 или первом post-MVP релизе?

### Тренировочная модель

- Нужны ли warm-up, working sets и failure sets?
- Нужны ли RPE/RIR?
- Нужны ли односторонние упражнения, где левая/правая сторона различаются?
- Нужны ли supersets или circuits?

### Единицы

- Какой шаг lb использовать по умолчанию?
- Нужно ли позже настраивать шаг веса?

### История и прогресс

- Как компактно показывать один прошлый результат?
- Нужны ли графики в MVP 1 или позже?
- Где разместить возврат к сегодняшнему дню?
- Какие точные цвета обозначают прошлое, сегодня и будущее?

### Бизнес

- Что долгосрочно значит “free”: без монетизации, бесплатное ядро с платными advanced features или корпоративная модель?
- Потребует ли будущая корпоративная интеграция организации, роли, админки или privacy boundaries сотрудников?
- Как должен выглядеть consent-based corporate sharing в продукте?
