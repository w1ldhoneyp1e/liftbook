# MVP 1 UX Polish Plan

Дата: 2026-04-30

Цель: закрыть UX-замечания перед MVP 1 без добавления новых продуктовых направлений. Этот документ написан так, чтобы следующая модель могла брать пункты по одному и реализовывать их без дополнительного обсуждения.

## 1. Горизонтальный скролл дат не должен менять день свайпом страницы

Проблема: при перетаскивании ленты дат на мобильном устройстве срабатывает общий свайп экрана и меняется выбранный день.

Решение:

- Перенести обработчик смены дня свайпом с общего `main` на отдельную область контента ниже sticky header.
- Либо останавливать touch propagation внутри ленты дат.
- Предпочтительно первое: свайп по списку дат должен скроллить список, свайп по контенту дня должен менять дату.

Файлы:

- `apps/web/src/features/day/day-screen.tsx`
- `apps/web/src/features/day/components/date-header.tsx`

Definition of Done:

- Горизонтальная лента дат скроллится пальцем.
- Выбранная дата не меняется при скролле самой ленты.
- Свайп по основному контенту дня продолжает менять дату.

## 2. Расширить количество дат в ленте

Проблема: сейчас в ленте мало дат.

Решение:

- Показывать 2 недели назад и 1 неделю вперед от выбранной даты.
- Проверить, что выбранная дата остается видимой.

Файлы:

- `apps/web/src/features/day/lib/date-utils.ts`

Definition of Done:

- В ленте есть даты примерно на 14 дней назад и 7 дней вперед.
- Нет горизонтального scrollbar в WebKit и Firefox.

## 3. Исправить оформление выбранной даты

Проблема: текущая тень выглядит как срезанный border, особенно сверху.

Решение:

- Убрать странную shadow-обводку.
- Использовать явный `border` или мягкий `ring` без визуального среза.
- Состояния `past / today / future` сохранить через цветовой тон.

Файлы:

- `apps/web/src/features/day/lib/date-utils.ts`
- `apps/web/src/features/day/components/date-header.tsx`

Definition of Done:

- Выбранная дата выглядит цельно.
- Верхняя часть обводки не кажется обрезанной.

## 4. Улучшить пустой день

Проблема: плейсхолдер "Добавить упражнение" выглядит бедно.

Решение:

- Центрировать пустое состояние.
- Убрать фон.
- Текст: `Добавьте первое упражнение на сегодня` для текущей даты.
- Для другой даты подставлять дату, например `Добавьте первое упражнение на 30.04`.
- Кнопка `Добавить` остается снизу через floating action button.

Файлы:

- `apps/web/src/features/day/components/exercise-list.tsx`
- `apps/web/src/shared/i18n/locales/ru.ts`
- `apps/web/src/shared/i18n/locales/en.ts`
- `apps/web/src/shared/i18n/dictionaries.ts`

Definition of Done:

- Пустой день выглядит легким и центрированным.
- Текст меняется в зависимости от выбранной даты.

## 5. Сделать старт и сброс таймера иконками

Проблема: текстовые кнопки `Старт` и `Сброс` занимают место в sticky header.

Решение:

- Использовать lucide icons: `Play`, `Pause`, `RotateCcw`.
- Оставить `aria-label` с переводимыми строками.

Файлы:

- `apps/web/src/features/day/components/rest-timer-row.tsx`

Definition of Done:

- Кнопки таймера компактные.
- Accessibility label остается.

## 6. Disable для "Свернуть все", если нет контента

Проблема: кнопка работает визуально даже когда упражнений нет.

Решение:

- Передать `disabled={exerciseEntries.length === 0}`.

Файлы:

- `apps/web/src/features/day/components/exercise-list.tsx`

Definition of Done:

- Кнопка disabled на пустом дне.

## 7. Calendar и Settings должны открываться сверху

Проблема: drawer появляется снизу, но пользователь ожидает его сверху для этих элементов.

Решение:

- Для `CalendarDrawer` и `SettingsDrawer` использовать `direction="top"` если текущий drawer-компонент это поддерживает.
- Если не поддерживает, расширить wrapper `Drawer` без поломки exercise picker.
- Exercise picker оставить снизу.

Файлы:

- `apps/web/src/components/ui/drawer.tsx`
- `apps/web/src/features/day/components/calendar-drawer.tsx`
- `apps/web/src/features/day/components/settings-drawer.tsx`

Definition of Done:

- Calendar и Settings открываются сверху.
- Exercise picker продолжает открываться снизу.

## 8. Позиционировать кнопку добавления упражнения относительно `main`

Проблема: floating button сейчас позиционируется относительно viewport.

Решение:

- Сделать `main` относительным контейнером.
- Перенести кнопку внутрь `main` и использовать absolute/fixed так, чтобы она визуально была привязана к мобильной ширине.
- На широком экране кнопка не должна уезжать за пределы app shell.

Файлы:

- `apps/web/src/features/day/day-screen.tsx`

Definition of Done:

- FAB визуально находится внутри мобильного app container.

## 9. Убрать черные пиксели по краю popup/drawer

Проблема: во всплывающих элементах видны черные пиксели по краю.

Решение:

- Проверить overlay/content border radius и overflow.
- Добавить `overflow-hidden` на drawer content при необходимости.
- Проверить, что фон overlay/content не просвечивает.

Файлы:

- `apps/web/src/components/ui/drawer.tsx`
- компоненты drawer при необходимости

Definition of Done:

- Края drawer выглядят чисто, без черных артефактов.

## 10. Успокоить визуальное состояние синхронизации

Проблема: статус дергается между `Все актуально` и `Синхронизация...`.

Решение:

- Не показывать `Синхронизация...` для тихого auto-pull, если нет pending changes.
- Либо добавить минимальное debounce/quiet mode для auto sync UI.
- Ручная синхронизация должна показывать прогресс явно.

Файлы:

- `apps/web/src/features/day/day-screen.tsx`
- `apps/web/src/features/day/components/settings-drawer.tsx`

Definition of Done:

- Пользователь не видит постоянное мерцание sync status.
- Ручной sync все еще дает обратную связь.

## 11. Убрать скролл из категорий при добавлении

Проблема: категории в exercise picker скроллятся горизонтально.

Решение:

- Перевести категории в wrapping chips.
- Ограничить высоту при необходимости.
- Поиск должен оставаться выше категорий.

Файлы:

- `apps/web/src/features/day/components/exercise-picker-drawer.tsx`

Definition of Done:

- Горизонтального скролла категорий нет.
- Категории удобно нажимать пальцем.

## 12. Исправить визуализацию focus на поиске

Проблема: autofocus на поиске выглядит странно.

Решение:

- Проверить `Input` focus ring внутри drawer.
- Если autoFocus вызывает визуальный шум, убрать `autoFocus` и оставить первый тап пользователю.
- Альтернатива: сохранить autofocus, но сделать focus ring мягче внутри exercise picker.

Файлы:

- `apps/web/src/features/day/components/exercise-picker-drawer.tsx`
- `apps/web/src/components/ui/input.tsx` только если нужно системное изменение.

Definition of Done:

- Поиск не выглядит сломанным при открытии drawer.

## 13. Красивее отобразить упражнения и подходы внутри дня

Статус: взято в текущую реализацию.

Решение:

- Сделать карточку упражнения компактной и спокойной, без внешнего бордера, тени и сворачивания.
- В header показывать название упражнения и группу мышц.
- Подходы показывать в одну линию компактными элементами.
- Вес и повторения редактировать во всплывающем редакторе по клику на подход.
- Добавление нового подхода делать отдельной кнопкой `+`.
- Кнопка `...` рядом с упражнением пока содержит только удаление упражнения.
- Удаление и редактирование подхода оставить в том же popover/popup, чтобы сам список был коротким.

Файлы:

- `apps/web/src/features/day/components/exercise-card.tsx`
- `apps/web/src/features/day/components/set-number-control.tsx`
- `apps/web/src/components/ui/popover.tsx`

Definition of Done:

- Карточка упражнения компактная.
- Подходы легко редактируются без перегруза основного экрана.

## 14. Убрать заголовок "Упражнения"

Проблема: заголовок `Упражнения` не добавляет ценности и только отнимает вертикальное место.

Решение:

- Удалить заголовок секции из основного контента.
- Сразу показывать список карточек упражнений или пустое состояние без дополнительного промежуточного заголовка.

Файлы:

- `apps/web/src/features/day/components/exercise-list.tsx`

Definition of Done:

- Основной контент начинается сразу со списка упражнений или пустого состояния.
- Вертикального шума в начале контента стало меньше.

## 15. Увеличить размеры основного контента

Проблема: основной контент страницы выглядит немного слишком мелким для быстрого использования во время тренировки.

Решение:

- Увеличить размер шрифта у ключевых элементов карточек упражнения и подходов.
- Чуть увеличить высоту и внутренние отступы у tappable-элементов в основном контенте.
- Не трогать sticky header без необходимости; фокус именно на рабочей области дня.

Файлы:

- `apps/web/src/features/day/components/exercise-card.tsx`
- `apps/web/src/features/day/components/set-number-control.tsx`
- при необходимости `apps/web/src/features/day/components/exercise-list.tsx`

Definition of Done:

- Карточки и подходы читаются легче с руки.
- Интерактивные элементы в основном контенте проще нажимать пальцем.
- UI остается mobile-first и не ломает ввод веса/повторений.

## 14. Безопаснее удалить упражнение и подход

Проблема: delete-кнопки сейчас слишком резкие.

Решение:

- Для удаления упражнения добавить confirmation alert/dialog.
- Для удаления подхода заменить trash на `X` и визуально сделать кнопку менее опасной.
- Если после пункта 13 delete action стал менее навязчивым, все равно добавить подтверждение для удаления всего упражнения.

Файлы:

- `apps/web/src/features/day/components/exercise-card.tsx`
- возможно новый shared confirm helper/dialog позже

Definition of Done:

- Удаление упражнения требует подтверждения.
- Удаление подхода визуально легче и менее похоже на опасное действие всей карточки.
